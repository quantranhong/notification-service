2. Send immediate notification (REST)

sequenceDiagram
    participant Client as 👤 Client
    participant API as REST / GraphQL
    participant Engine as Notification Engine
    participant PG as PostgreSQL
    participant Redis as Redis
    participant Channel as SMS/Email/Push Provider
    participant MDB as MongoDB

    Client->>+API: POST /api/v1/notifications (channel, audience, templateId, context)
    API->>API: Validate, resolve template
    API->>+Engine: sendToAudience(...)
    Engine->>PG: Resolve recipients (internal_users or subscriptions)
    PG-->>Engine: List of destinations
    loop For each recipient
        Engine->>Redis: Check idempotency key (if provided)
        alt Cache hit
            Redis-->>Engine: Cached result
        else Cache miss
            Engine->>Engine: Render body/subject from template + context
            Engine->>Channel: send(to, subject, body)
            Channel-->>Engine: DeliveryResult
            Engine->>MDB: Append NotificationEvent (audit)
            Engine->>Redis: Cache result (idempotency)
        end
    end
    Engine-->>-API: List of DeliveryResult
    API-->>-Client: 200 OK + successCount, results

3. New mobile user subscribed → email to internal users

sequenceDiagram
    participant App as 📱 App / Backend
    participant API as NewSubscriberController
    participant Engine as Notification Engine
    participant PG as PostgreSQL
    participant Email as Email Provider
    participant MDB as MongoDB

    App->>+API: POST /api/v1/events/mobile-user-subscribed
{ userId, email } + templateId + Idempotency-Key
    API->>+Engine: sendToAudience(templateId, EMAIL, INTERNAL, context, idempotencyPrefix)
    Engine->>PG: internalUserRepository.findAll()
    PG-->>Engine: Internal user emails (e.g. operators, support)
    loop For each internal user
        Engine->>Engine: renderBody / renderSubject (e.g. "New subscriber {{userId}}")
        Engine->>Email: send(to, subject, body)
        Email-->>Engine: DeliveryResult
        Engine->>MDB: Save NotificationEvent
    end
    Engine-->>-API: results
    API-->>-App: 202 Accepted

4. Scheduled push/notifications (Scheduler Service → Notification Service)

sequenceDiagram
    participant Cron as ⏰ Cron (every minute)
    participant Scheduler as Scheduler Service (Quarkus)
    participant NS as Notification Service
    participant Internal as InternalProcessController
    participant SchedSvc as ScheduledNotificationService
    participant Engine as Notification Engine
    participant PG as PostgreSQL
    participant Channel as SMS/Email/Push Provider
    participant MDB as MongoDB

    Cron->>+Scheduler: Trigger
    Scheduler->>NS: POST /api/v1/internal/process-due-scheduled
Header: X-Internal-Api-Key
    NS->>+Internal: processDueScheduled()
    Internal->>Internal: Validate API key (if configured)
    Internal->>+SchedSvc: processDueScheduled()
    SchedSvc->>PG: findDueScheduled(PENDING, now)
    PG-->>SchedSvc: List of ScheduledNotification
    loop For each due scheduled
        SchedSvc->>PG: Update status → SENT
        SchedSvc->>+Engine: sendToAudience(templateId, channel, audienceType, ...)
        Engine->>PG: Resolve recipients
        Engine->>Channel: Send to each recipient
        Engine->>MDB: Log event
        Engine-->>-SchedSvc: results
    end
    SchedSvc-->>-Internal: done
    Internal-->>-NS: 202 Accepted
    NS-->>-Scheduler: Response
    Scheduler-->>-Cron: Done
